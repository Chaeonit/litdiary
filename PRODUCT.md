# LinguaDiary — 서비스 기획 정리

> "언어 학습은 꾸준함이 전부다. 하지만 꾸준하려면 즐거워야 한다."
> LinguaDiary는 '일기'라는 익숙한 행위를 언어 학습의 진입점으로 삼는다.

---

## 1. 핵심 아이디어

| 질문 | 답 |
|------|---|
| 누가 쓰는가? | 영어 또는 중국어를 학습 중인 한국인 |
| 무엇을 하는가? | 매일 외국어로 일기를 쓰고 AI 피드백을 받는다 |
| 왜 이게 효과적인가? | 자신의 경험을 쓰기 때문에 맥락이 있고, 오류가 본인 글에서 나오기 때문에 기억에 남는다 |
| 기존 서비스와 다른 점 | 문법 체커가 아니라 '내 일기'를 기반으로 한 맞춤 학습 루프를 제공한다 |

---

## 2. 사용자 여정 (User Journey)

```
① 오늘의 일기 작성
   └─ 언어 선택 (영어 / 중국어)
   └─ 자유롭게 텍스트 입력

② AI 피드백 (Structured Output)
   └─ 교정된 문장 제시
   └─ 오류 하이라이트 + 한국어 설명
   └─ 점수(0-100) + 레벨(초급/중급/고급) + 총평

③ 추가 학습 (Function Calling)
   └─ 빈칸 채우기 퀴즈 (내 오류 기반으로 생성)
   └─ 관련 단어 카드 (예문 + 유사 표현 포함)

④ 저장 & 기록
   └─ 일기 저장 → 히스토리 탭에서 복습 가능
   └─ 점수 추이로 성장 체감
```

---

## 3. 유저 경험 설계 원칙

### 3-1. 부담 없이 시작하게 한다
- 일기는 완벽하지 않아도 된다. 틀리는 게 당연하고, 그게 학습 재료가 된다.
- "틀렸다"는 느낌보다 "이렇게 쓰면 더 자연스럽다"는 어투로 피드백 전달.

### 3-2. 내 글에서 출발하는 학습
- 퀴즈와 단어는 외부 데이터가 아니라 **내가 쓴 문장**에서 파생된다.
- 맥락이 있기 때문에 단순 단어 암기보다 훨씬 잘 기억된다.

### 3-3. 한 사이클이 3분 안에 끝나야 한다
- 분석 요청 → 결과 확인 → 퀴즈 → 저장 흐름이 3분 이내로 완결되어야 한다.
- 무겁고 긴 학습보다 **짧고 자주**가 핵심.

### 3-4. 성장이 보여야 한다
- 점수와 레벨이 기록에 남아 "지난 달보다 나아졌다"를 체감할 수 있어야 한다.

## 4. 핵심 기술과 역할

| 기술 | 역할 | 왜 이 방식인가 |
|------|------|---------------|
| **OpenRouter API** | AI 모델 호출 인터페이스 | 단일 API로 다양한 모델 선택 가능, 비용 효율적 |
| **Structured Output** | 문법 분석 결과를 JSON으로 수신 | 오류 목록·점수·레벨을 UI에 바로 렌더링하기 위해 구조 보장 필요 |
| **Function Calling** | 퀴즈·단어장 생성을 AI가 능동적으로 수행 | AI가 분석 결과를 보고 어떤 학습 콘텐츠가 필요한지 스스로 판단 |

---

## 5. Structured Output 설계

```json
{
  "corrected_text": "교정된 전체 문장",
  "errors": [
    {
      "original": "I goed to school",
      "corrected": "I went to school",
      "type": "grammar",
      "explanation_kr": "go의 과거형은 불규칙 변화하여 went입니다."
    }
  ],
  "score": 78,
  "level": "intermediate",
  "overall_feedback_kr": "전반적으로 문장 구조는 좋으나 불규칙 동사 활용에 주의가 필요합니다."
}
```

**strict 모드를 사용하는 이유:** UI 렌더링이 필드 존재 여부에 의존하기 때문에 없는 필드가 있으면 화면이 깨진다. strict 모드로 필드 누락을 방지한다.

---

## 6. Function Calling 설계

AI가 일기 분석 후 아래 두 함수를 **동시에 호출**하도록 유도한다.

### `generate_fill_in_blank_quiz`
- 오류가 발생한 문법 패턴을 반복 연습하는 문제 생성
- 3~4문항, 각 문항에 한국어 힌트 포함

### `provide_related_vocabulary`
- 일기 주제와 오류에 연관된 단어 4~5개 선정
- 단어 + 한국어 뜻 + 예문 + 유사 표현

**Function Calling을 쓰는 이유:** 추가 학습 콘텐츠 생성을 단순 텍스트 응답이 아닌 함수 호출로 처리하면, AI가 어떤 학습이 필요한지 **능동적으로 판단**하는 구조를 만들 수 있다. 또한 결과가 정형화된 JSON으로 오기 때문에 UI 조립이 깔끔하다.

## 7. 서비스 흐름을 유연하게 만드는 요소

### 지금 구현된 것 (MVP)
- [x] 텍스트 일기 작성 (영어 / 중국어)
- [x] AI 문법·어휘 분석 (Structured Output)
- [x] 빈칸 채우기 퀴즈 (Function Calling)
- [x] 관련 단어 학습 카드 (Function Calling)
- [x] 일기 저장 및 히스토리 조회

### 다음 단계로 확장 가능한 것
- [ ] **음성 녹음** — Web Speech API + Whisper API로 음성 → 텍스트 변환 후 동일 흐름
- [ ] **단어장 누적** — 단어 카드를 DB에 저장해서 별도 복습 모드 구현
- [ ] **주간 리포트** — 한 주간 점수 추이 + 반복 오류 패턴 요약
- [ ] **레벨별 가이드** — 초급자에게는 더 쉬운 피드백, 고급자에게는 뉘앙스까지 설명
- [ ] **소셜 기능** — 익명으로 일기 공유, 원어민 피드백 받기

## 8. 경쟁 서비스 비교

| 서비스 | 방식 | LinguaDiary와 차이 |
|--------|------|-------------------|
| Duolingo | 게임형 단어/문법 학습 | 내 언어가 아닌 외부 콘텐츠 기반 |
| Grammarly | 문법 교정 도구 | 학습보다 교정에 집중, 추가 학습 없음 |
| HelloTalk | 원어민 대화 | 실시간 사람 의존, 부담이 큼 |
| **LinguaDiary** | 내 일기 기반 AI 맞춤 학습 | 자유로운 쓰기 + 즉시 피드백 + 맞춤 퀴즈 |

## 9. 성공 지표 (KPI 아이디어)

- DAU 일기 작성 수 (꾸준함 측정)
- 퀴즈 정답률 추이 (학습 효과 측정)
- 평균 점수 변화 (성장 체감)
- 히스토리 탭 방문율 (복습 행동 측정)
